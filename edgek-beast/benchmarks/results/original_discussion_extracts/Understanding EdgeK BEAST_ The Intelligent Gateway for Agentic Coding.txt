Tool Tax:  The massive recurring overhead of injecting full MCP tool schemas and descriptions into every single request, regardless of whether the agent needs them.

2. What is EdgeK BEAST? The Governed Agentic Execution Broker
EdgeK BEAST  (Budgeted Execution and Agentic Systems Tunnel) is a local-first, policy-governed control plane designed to sit between coding clients and LLM providers. Rather than a "stealth" bypass, BEAST is an  Authorized Efficiency  platform. It ensures that every token spent is an investment in reasoning, not a tax on infrastructure.BEAST is defined by three fundamental architectural design goals:
Protocol Unification via Request IR:  BEAST acts as a universal translator, converting all inbound traffic from various providers (Anthropic, OpenAI, Gemini) into a  Request Intermediate Representation (IR) . This normalized envelope allows the gateway to apply logic consistently regardless of the backend.
Governed Context Economy:  It treats the context window as a finite resource, employing structural compression and indexing to ensure the model receives only high-signal data.
Runtime Control Plane:  It functions as a safety and pacing officer, using "Leaky-Bucket" logic to manage rate limits and prevent agents from entering expensive failure loops.By enforcing these goals, BEAST transforms a chaotic stream of text into a governed, observable engineering workflow.

5. The Context Economizer: Cutting the "Token Tax"
Traditional "regex stripping" is a dangerous anti-pattern in coding; deleting a critical comment or linter directive can lead to broken edits. BEAST uses a  Three-Tier Context Economy Pipeline  to maintain code integrity:
Lossless Cleanup:  Removal of ANSI logs, trailing whitespace, and "folding" of duplicate log lines.
Structure-Aware Compression:  Using  Tree-sitter summaries  and AST-based symbol maps to provide the model with "sketches" of files rather than raw text.
Semantic Compression:  Prompt-aware retrieval that only serves relevant code "slices" anchored to specific line ranges.
Safe Compression Rules
The gateway adheres to strict heuristics to ensure the agent never loses the "plot" of the codebase:
Preserve failing assertions and the first error cause:  Never hide the diagnostic data the agent needs to fix a bug.
Never remove comments globally:  Comments often house API contracts and security warnings.
Keep active edit targets verbatim:  The specific file the agent is modifying must remain uncompressed for accuracy.
Maintain line-number anchors:  Essential for ensuring the agent's "diff" applications land correctly.

7. Governed MCP: Turning Tools from Toys into Assets
The Model Context Protocol (MCP) is powerful but creates a massive "Tool Tax." BEAST acts as a  Governed MCP Broker , employing  Schema Pinning and Hashing  to prevent "tool poisoning"—a security threat where tool descriptions are maliciously altered.BEAST manages tool exposure through three modes:
Static Mode:  A fixed, minimal set of approved tools for simple tasks.
Lazy Mode:  The gateway only reveals tool schemas to the model as they become relevant to the task intent.
Search Mode:  The model must explicitly "ask" for a tool by name before the gateway provides the instructions.This "Tool-Laziness" reduces the schema token count by up to 70%, keeping the prompt lean and secure.
